C语言实现TCP连接

时间  2015-07-26
栏目 系统网络

这两天复习了不少有关Windows平台和Linux平台的Socket编程知识，之前曾经写过一篇博文，示范过怎么使用C语言实现TCP连接，但是那篇文章写得很随意，所以现在我决定要重写这篇文章，既是为了总结这两天学到的知识，也为了给日后我写代码一个参考。

下面我用给出两份C语言代码来实现TCP连接，采用边写代码边解释的办法讲解这些程序，越复杂的代码越难看懂，因此我将给出最简洁的代码、包含最少的头文件来实现一个只发出一句话就随即关闭的程序。

 

首先总结一下TCP的连接过程：

服务端：

  1. 创建一个Socket    「 socket() 」

  2. 创建并初始化本机地址结构体    「 struct sockaddr_in 」

  3. 绑定Socket和地址结构体     「 bind() 」

  4. 监听连接     「 listen() 」

  5. 接受连接     「 accept() 」

  6. 开始自由通信     「 Linux: read()/write() 」 「 Windows: recv()/send() 」

  7. 关闭Socket      「 close() 」

     

客户端：

  1. 创建一个Socket     「 socket() 」

  2. 创建并初始化服务器地址结构    「 struct sockaddr_in 」

  3. 连接服务器      「 connect() 」

  4. 开始自由通信        「 Linux: read()/write() 」「 Windows: recv()/send() 」

  5. 关闭Socket          「 close() 」

 

代码：

服务端(Windows平台)：

#include <stdio.h>
#include <winsock2.h>
int main(){
    //启动Socket前的初始化
    WSADATA wsaData;
    WSAStartup(MAKEWORD(2, 2), &wsaData);
    /*在windows平台上使用Socket编程前 *必须* 先调用WSAStartup()函数初始化一些工作，
        是关于ws2_32.dll调用的初始化,ws2_32.dll提供了实现网络连接必要的函数，
        当程序运行结束以后需要调用WSACleanup()来清理、释放资源，
        如果以linux为服务器，则不需要这项工作*/

    //创建SOCKET
    SOCKET serverSocket = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    /*在一次通信中，通信的双方每边都会有一个SOCKET，用于标识自己
        SOCKET这东西在我看来，就跟一个文件流没什么差别，
        在linux上我们可以调用write()向要通信的对方的SOCKET中写入数据，
        对方可以调用read()从他自己的SCOKET中读取数据我们刚发送的数据，
        就像操作一个普通文件一样，只不过这个"文件"是储存在另一台电脑上而已，
        所以把它理解成一个通信的通道就好了*/
    
    //绑定本机的地址到SOCKET
        //首先需要构造一个地址结构体来储存我们自己的IP和端口
        //这个结构体的具体描述在网上有很多介绍，也可以打开winsock2.h头文件查看详细信息，这里就不多说了
    struct sockaddr_in serverAddr;
    serverAddr.sin_family = AF_INET;    //TCP协议这么写就好了
    serverAddr.sin_port = htons(1187);    //端口号，htons()这个函数大家有必要去了解一下
    serverAddr.sin_addr.s_addr = INADDR_ANY;  //本机的IP地址，上次我在这里填了127.0.0.1，
                                        //结果局域网里的其他机器都连接不上，于是改成了这样才可以
        //这样就把本机的IP和SOCKET绑定起来了
    bind(serverSocket,(struct sockaddr*)&serverAddr,sizeof(serverAddr));
    
    //设置监听参数
    listen(serverSocket,1);
    /*第二个参数的意思不是最多能接受多少个连接，而是同时能处理几个主机的连接请求，
        这里设置为1，假设服务器上线以后，同时有两台主机请求连接，
        那么就最多只会处理一个连接，而另一台主机只能连接失败
        值得注意的是，程序运行到这里并不会停止下来，监听端口，等待连接。
        我之前一直以为服务器等待连接是在这里发生的，真正等待连接的函数是下面的accept() */
    
    //接受连接
    struct sockaddr_in clientAddr;
    int clientAddrSize = sizeof(struct sockaddr_in);
    SOCKET clientSocket = accept(serverSocket, (struct sockaddr*)&clientAddr, &clientAddrSize);
    /*这只是个简单示例，所以只用了一个客户SOCKET，也就是说只要有一台主机连接进来，
        我们就不会再处理新的连接，而真实的服务器一般都是用多线程来接受客户连接的，
        每接收到一个新的连接，就创建一个新线程来跟客户端交互。
        程序运行到accept()函数这里就会阻塞住，直到有新的连接才会继续运行下去，
        accpet()函数需要一个sockaddr_in结构图作为参数，
        函数调用完毕后这个结构体里就储存着客户端的IP地址，
        accpet()函数返回的是新连接的SOCKET,我们下面跟对方交互就用这个SOCKET。*/
    
    //向客户端发送数据
    char str[] = "Hello,friend!";
    send(clientSocket, str, sizeof(str), 0);
    /*发送数据很简单，参数基本都能理解，最后一个参数我还没有仔细研究过，只知道一般置0就好了*/
    
    //关闭套接字
    closesocket(serverSocket);
    closesocket(clientSocket);
    
    //首尾呼应
    WSACleanup();
    return 0;
}

客户端(Linux平台)：

#include <stdio.h>
#include <sys/socket.h>
#include <netinet/in.h>

int main(){

  //构造socket
  int mySocket = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
  printf("Socket: %d\n",mySocket);

  //连接服务器，用的是connect()函数，调用需要一个sockaddr_in结构体
  struct sockaddr_in serverAddr;
  serverAddr.sin_family = AF_INET;
  serverAddr.sin_port = htons(1187);
  serverAddr.sin_addr.s_addr = inet_addr("192.168.0.XXX");  //这里填上服务器的IP地址
  connect(mySocket, (struct sockaddr*)&serverAddr, sizeof(struct sockaddr));
  
  //接收服务器发送的数据
  char buffer[40];
  read(mySocket, buffer, sizeof(buffer)-1);
  printf("Message form server: %s\n", buffer);

  //关闭连接
  close(mySocket);
  
  return 0;
}
